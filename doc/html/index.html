Om libStartrail ver 2.1.6.

Author:
fredrik persson<fpersson.se@gmail.com>

Allmänt

libStartrail är i huvudsak skrivit för mitt eget höga nöjesskull för att testa att binda Lua mot C++. Men det finns en praktisk nytta bakom projektet om man gillar att "stacka" i hop en jävla massa bilder

Beroenden.

  • Magick++
  • Boost
  • Lua 5.1
  • Exiv2
  • Pyton (och boost::python)

Nytt 2.1.6:

  • Kan nu även bygga för python, med boost::python, libfilen blir 1.3MB stor jämfört med Lua versionen som blir ca 600kb

Tidigare ändringar (se News)

Bygg.

Det nya byggsystemet med cmake gör att man kan bygga både libstartrail (Lua/Python versioner) och startrail, eller varje del för sig beroende på vad som önskas.

  • cmake ./
  • make (bygger ./bin/startrail, ./lib/libstartrail samt ./python/lib_pystartrail)
  • make -C ./bin (bygger bara ./bin/startrail)
  • make -C ./lib (bygger bara ./lib/libstartrail)
  • make -C ./python (bygger bara ./python/lib_pystartrail
  • make doc (skapar dokumentation från doxygen)

Anändning.

startrail.

Startrail version 2.1.x Generar en startrail bild utifrån en bildsammling.

Syntax: ./startrail --input-file=./images/test.txt --dest-file=./minbild.jpg

Tillåta argument:

  • --help - Hjälp.
  • --version - skriver ut versionsnumer
  • --input-file arg - Anger en textfil med en lista av bildfiler.
  • --dest-file arg - Anger målfilen som jpg.
  • --use-infofile arg - ange

libstartrail.

I från lua kan man använda två funktioner, för närmare beskrivning se i ./demo.lua under exempel.

  • ComposeFromFile, funktionen tar textfil som argument, textfilen förväntas innehålla filnamnen på de filer som ska användas.
  • ComposeFromArray, funktionen tar en array med filnamn.

lib_pystartrail.

Bara för att experimentera portade jag libstartrail till Python också med hjälp av boost::python
